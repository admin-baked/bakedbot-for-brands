## Creating Your First Pricing Rule

### Step 1: Choose a Strategy

**4 pricing strategies:**

1. **Clearance** - Move aging inventory before expiration
2. **Competitive** - Match or beat competitor prices
3. **Time-Based** - Happy hour, weekend specials, seasonal promos
4. **Dynamic** - Category-specific or segment-based pricing

### Step 2: Set Conditions

**Condition Types:**

| Condition | When to Use | Example |
|-----------|-------------|---------|
| **Inventory Age** | Clear old stock | Products 45+ days old |
| **Competitor Price** | Stay competitive | When 15%+ above market avg |
| **Time of Day** | Drive off-peak traffic | Weekdays 2-5pm |
| **Day of Week** | Weekend specials | Friday-Sunday |
| **Category** | Promote specific products | All flower products |
| **Stock Level** | Urgent clearance | Less than 5 units |
| **Traffic Level** | Surge pricing | Peak hours (controversial!) |

<Callout type="warning">
**Multiple conditions = AND logic:** All conditions must be met for the rule to apply. Use multiple rules if you need OR logic.
</Callout>

### Step 3: Configure Price Adjustment

**Percentage Discount** (recommended):
- Set discount: 10%, 20%, 30%, etc.
- Min price: Prevent selling below cost ($5.00 minimum)
- Max discount: System enforces 40% maximum

**Fixed Discount**:
- Dollar amount off: $5 off, $10 off, etc.
- Use for specific promotions

### Step 4: Set Priority

**Priority determines rule order (100 = highest):**

- **90-100**: Urgent clearance (expiring soon)
- **70-89**: Competitor matching (market-driven)
- **50-69**: Time-based promotions (happy hour, weekends)
- **20-49**: Category deals (ongoing)
- **1-19**: Default discounts (loyalty, new customers)

**Why priority matters:**
Only the **highest priority matching rule** applies to each product. If multiple rules match, the one with the highest priority wins.

## Ezal Competitor Intelligence

**What is Ezal?**
Ezal is BakedBot's competitive intelligence agent that scrapes competitor dispensary menus and tracks pricing.

### Enabling Competitor Data

1. Contact support to enable Ezal for your market
2. Provide list of competitor dispensaries to track
3. Wait 24-48 hours for initial data scraping
4. Competitor prices appear in Intelligence tab
5. Enable competitive pricing rules

### Competitor Price Conditions

**Above market:**
```
Competitor Price: above 15%
```
Apply discount when your price is 15%+ higher than competitor average.

**Below market:**
```
Competitor Price: below 10%
```
Reduce discount when your price is already 10%+ lower (maintain margin).

   - Triggers clearance rules when thresholds met

   - When rule applies, discount syncs to Alleaves POS

- Batch tracking enabled (for inventory age)
- API access configured
- Location ID set in integration settings

## Safety & Constraints

**Built-in protections:**

### 40% Maximum Discount
System enforces 40% max discount to prevent margin erosion. Even if you set 60% off, it will cap at 40%.

### Minimum Price Enforcement
Products will never sell below your configured minimum price (recommended: $5.00).

### Single Rule Application
Only **one rule** applies per product (highest priority). No stacking discounts.

### Cache & Performance
5-minute caching on inventory and competitor data prevents excessive API calls.

## FAQ

**Q: Can I stack multiple discounts?**
A: No, only one rule applies per product (highest priority). Create a single rule with combined discount if needed.

**Q: Will surge pricing upset customers?**
A: Yes, probably. Use surge pricing (negative discounts) very carefully. Customers dislike paying more during peak hours.

**Q: How often does pricing update?**
A: Real-time for rule changes, 5-minute cache for inventory/competitor data, instant sync to POS.

**Q: Can I override a rule for specific products?**
A: Yes, use product ID conditions or disable the rule temporarily. Manual POS discounts override dynamic pricing.

**Q: Do I need Ezal for dynamic pricing?**
A: No, Ezal is only required for competitor price matching rules. Time-based, clearance, and category rules work without it.

**Q: What if I don't have Alleaves POS?**
A: Dynamic pricing requires POS integration for inventory age tracking. We support Alleaves, Dutchie, and Jane. Contact support for other systems.
