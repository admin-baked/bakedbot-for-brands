## Performance Metrics

**Track agent performance in real-time:**

### Context Window
- **Messages Loaded:** Number of messages in current conversation
- **Tokens Used:** Total tokens consumed (out of 100k limit)
- **Utilization:** Percentage of context window used
- **Memory Retrievals:** Number of memory lookups performed

**Why it matters:** High utilization (>80%) means context is nearly fullâ€”agent may need memory gardening.

### Cognitive Load
- **Idle:** No active tasks
- **Light:** 1-2 tool calls, 0-1 queued requests
- **Moderate:** 3-5 tool calls, 2-3 queued requests
- **Heavy:** 6-10 tool calls, 4-5 queued requests
- **Overloaded:** 10+ tool calls, 5+ queued requests

**Why it matters:** Overloaded agents have slower response timesâ€”consider load balancing.

### Response Confidence
- **Current:** Confidence score (0.00-1.00) of last response
- **Average:** Rolling 30-day average confidence
- **Source Breakdown:** Where confidence comes from (memory, POS data, inference, etc.)

**Why it matters:** Low confidence (<0.60) suggests agent needs more training data or fact verification.

### Completeness Score
- **Score:** Percentage (0-100%) of user intents addressed
- **Missed Intents:** Number of questions left unanswered
- **Trend:** 7-day rolling average

**Why it matters:** Low completeness (<70%) means agent is missing user questionsâ€”investigate intent extraction.

## Best Practices

### 1. **Match Mode to Task**
Don't use Technical mode for customer-facing content or Casual mode for investor reports.

### 2. **Monitor Confidence**
If average confidence drops below 0.70, investigate:
- Are agents accessing the right data sources?
- Is ground truth up to date?
- Do agents need retraining?

### 3. **Balance Cognitive Load**
If an agent is consistently **Heavy** or **Overloaded**:
- Delegate tasks to other agents
- Reduce tool call frequency
- Optimize workflows

### 4. **Document Custom Configurations**
When you find a winning slider configuration, save it:
- Take a screenshot
- Note the use case
- Share with team

### 5. **Review Completeness Weekly**
Low completeness means customers aren't getting full answers:
- Check **Completeness Score** tab
- Review missed intents
- Adjust agent prompts or grounding

## Troubleshooting

### Agent Status Shows "Error"
**Cause:** Agent encountered a critical failure.

**Fix:**
1. Check **CEO Dashboard â†’ Intelligence â†’ System Logs**
2. Review error messages
3. Contact support if unresolved

### Sliders Not Affecting Behavior
**Cause:** Slider changes apply to new requests, not in-progress conversations.

**Fix:**
1. Start a new conversation in Inbox
2. Test with a fresh request
3. Verify slider values saved

### High Context Utilization (>90%)
**Cause:** Context window nearly fullâ€”agent may hallucinate or forget details.

**Fix:**
1. Run **Memory Gardening** (see Memory Health guide)
2. Clear old conversation threads
3. Reduce message history retention

### Low Confidence Scores (<0.60)
**Cause:** Agent lacks reliable data sources or grounding.

**Fix:**
1. Update **Ground Truth** with authoritative facts
2. Sync POS data (products, inventory, pricing)
3. Verify integrations (Alleaves, Dutchie, Jane)
